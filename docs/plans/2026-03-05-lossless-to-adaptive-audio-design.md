# losslessToAdaptiveAudio 设计文档

- 日期：2026-03-05
- 目标脚本：`scripts/pwsh/misc/losslessToQaac.ps1`（将重命名为 `scripts/pwsh/misc/losslessToAdaptiveAudio.ps1`）

## 背景

现有脚本仅依赖 `qaac.exe` 进行编码，若本机未安装 `qaac`，脚本无法工作。需要在 `qaac` 不存在时自动降级为 `ffmpeg libopus` 编码，以保证可用性。

## 已确认需求

1. 仅当 `qaac.exe` 不存在时回退到 `ffmpeg`，不在 `qaac` 执行失败时自动回退。
2. 回退模式输出容器使用 `.ogg`。
3. 回退模式编码参数固定为 `-c:a libopus -b:a 256k`。
4. 脚本文件名改为 `losslessToAdaptiveAudio.ps1`。

## 方案对比

### 方案 A：启动时一次性选择编码器（推荐）

- 启动时探测 `qaac`，若不存在则探测 `ffmpeg` 并切换到 `libopus`。
- 后续所有文件在同一策略下处理。
- 优点：实现简单、行为稳定、改动最小。
- 缺点：不处理“qaac 存在但个别文件失败时自动切换”。

### 方案 B：按文件动态选择编码器

- 每个文件处理前都探测可用编码器。
- 优点：灵活。
- 缺点：复杂度高、日志噪声大、收益有限。

### 方案 C：抽象完整编码器策略层

- 重构为可扩展多编码器框架。
- 优点：长期扩展性好。
- 缺点：对当前需求过度设计。

最终采用：方案 A。

## 详细设计

### 架构与组件

- 保留现有并行处理主流程（`ForEach-Object -Parallel`）。
- 在并行前新增“编码器策略选择”：`qaac` 或 `ffmpeg-opus`。
- 通过 `$using:` 将策略与必要参数传入并行块。

### 数据流

1. 收集无损文件（`.flac`、`.wav`）。
2. 启动时探测：
   - `qaac.exe` 可用：使用现有 AAC 路径。
   - `qaac.exe` 不可用：要求 `ffmpeg` 可用，否则终止并提示安装依赖。
3. 并行编码：
   - `qaac` 模式：保持现有命令逻辑，输出 `.m4a`。
   - `ffmpeg-opus` 模式：执行 `ffmpeg -y -i <input> -c:a libopus -b:a 256k <output.ogg>`。
4. 输出文件存在时，按现有逻辑决定是否删除源文件（受 `-nodelete` 与 `-WhatIf` 控制）。

### 参数与兼容策略

- `-he` 仅在 `qaac` 模式生效；`ffmpeg-opus` 模式忽略并提示一次。
- `-WhatIf` 语义保持：仅打印将执行命令，不实际编码或删除。
- 现有并发、进度输出、异常捕获风格保持不变。

### 错误处理

- `qaac` 与 `ffmpeg` 均不可用：启动即终止，明确错误提示。
- 单文件编码失败：记录错误并继续后续文件，不中断整体流程。
- 若目标文件未生成：报告失败并保留源文件。

## 验证计划

1. `qaac` 可用环境：确认仍输出 `.m4a`，行为与原脚本一致。
2. `qaac` 不可用且 `ffmpeg` 可用：确认输出 `.ogg`，命令含 `libopus` 与 `-b:a 256k`。
3. `-WhatIf`：确认只打印命令与删除预演信息，不产生文件变更。
4. `-nodelete`：确认目标文件存在时源文件不删除。
5. 任务结束执行 `pnpm qa` 并修复检查问题。

## 风险与取舍

- 回退到 Opus 后输出格式不再是 AAC（从 `.m4a` 变为 `.ogg`），是有意行为变更。
- 不在 `qaac` 失败时自动回退，简化逻辑并严格贴合需求。
