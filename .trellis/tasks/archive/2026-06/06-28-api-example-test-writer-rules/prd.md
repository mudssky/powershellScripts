# 规范 API 示例测试编写 skill

## Goal

更新 `ai/skills/dev/api-example-test-writer` 的编写约定，让它生成的 httpyac 活示例更适合人工阅读与调试：少抽不必要的环境变量，断言不遮挡响应内容，日期时间参数保持可读。

## Requirements

- 环境变量只用于密码、token、client secret、base URL、账号/租户等机器差异值，或会在多个请求中复用的稳定值；一次性查询参数、普通筛选条件和只出现一次的业务值优先直接写在请求里。
- httpyac 请求首先是可读活示例。自动断言应保持轻量，不得让人发送请求后只看到测试结果而看不到原始响应内容。
- 业务失败检查可以存在，但应优先作为可选/辅助校验，并在文档中提示如何保留响应可见性。
- 日期、日期时间和时间戳参数优先用人能读懂的表达方式，例如 ISO 日期字符串或 `Date.parse('2024-04-01T00:00:00+08:00')`；只有接口明确要求纯时间戳时才转换为数值。
- 更新内容应落在现有 skill 文档和 httpyac 参考中，不新增无关说明文件。

## Acceptance Criteria

- [x] `SKILL.md` 明确环境变量抽取边界，避免把一次性参数都抽进 env。
- [x] `references/httpyac-patterns.md` 明确响应可见性优先，给出轻量断言与可选深度校验的写法取舍。
- [x] 文档中包含日期时间可读表达式规则，并覆盖 `Date.parse(...)` 这类时间戳转换场景。
- [x] 示例或参考内容不再暗示所有查询参数都应该环境变量化。
- [x] 修改范围保持在 `api-example-test-writer` 相关文件和 Trellis 任务文件内。
