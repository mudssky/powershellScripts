
这份 Cheatsheet 将展示如何构建一个**工业级、高容错、易维护**的 API 客户端

### 📂 推荐目录结构

```text
my_integration/
├── .env                    # 环境变量
├── config.py               # pydantic-settings 配置管理
├── schemas.py              # pydantic 数据模型 (DTO)
├── client/
│   ├── __init__.py         # 暴露统一入口
│   ├── core.py             # 核心基类 (httpx + tenacity + loguru)
│   └── services/           # 业务接口拆分
│       ├── user.py
│       └── order.py
└── main.py                 # 调用示例
```

---

### 1. 配置层 (`config.py`)

**工具**: `pydantic-settings`
**作用**: 从环境变量加载配置，强类型校验，避免 `os.getenv` 散落在各处。

```python
from pydantic_settings import BaseSettings, SettingsConfigDict

class Settings(BaseSettings):
    API_BASE_URL: str = "https://api.example.com"
    API_TIMEOUT: int = 10
    API_TOKEN: str

    # 自动读取 .env 文件
    model_config = SettingsConfigDict(env_file=".env", env_file_encoding="utf-8")

settings = Settings()
```

---

### 2. 数据模型层 (`schemas.py`)

**工具**: `pydantic`
**作用**: 定义接口的入参和出参，提供代码提示和自动校验。

```python
from pydantic import BaseModel, Field, EmailStr
from typing import List, Optional

# --- Response Models ---
class UserDTO(BaseModel):
    id: int
    username: str
    email: Optional[EmailStr] = None

class UserListDTO(BaseModel):
    items: List[UserDTO]
    total: int

# --- Request Models ---
class CreateUserRequest(BaseModel):
    username: str = Field(..., min_length=3)
    password: str
```

### 3. 核心基类 (`client/core.py`)

**工具**: `httpx`, `tenacity`, `loguru`
**作用**: 封装所有通用逻辑（重试、日志、鉴权、异常处理）。这是系统的“心脏”。

```python
import httpx
from tenacity import retry, stop_after_attempt, wait_fixed, retry_if_exception_type
from loguru import logger
from typing import Any, Dict, Type, TypeVar, Optional
from pydantic import BaseModel
from config import settings

T = TypeVar("T", bound=BaseModel)

class BaseClient:
    def __init__(self):
        self.client = httpx.Client(
            base_url=settings.API_BASE_URL,
            timeout=settings.API_TIMEOUT,
            headers={"Authorization": f"Bearer {settings.API_TOKEN}"}
        )

    # --- Tenacity 重试装饰器配置 ---
    # 仅在发生网络错误或特定 HTTP 错误时重试，重试 3 次，间隔 2 秒
    @retry(
        stop=stop_after_attempt(3),
        wait=wait_fixed(2),
        retry=retry_if_exception_type(httpx.RequestError),
        reraise=True
    )
    def _request(self, method: str, endpoint: str, **kwargs) -> Dict[str, Any]:
        """底层请求封装：包含日志、重试、错误抛出"""
        url = f"{settings.API_BASE_URL}{endpoint}"

        # 1. 记录请求日志
        logger.info(f"Requesting: {method} {url} | Params: {kwargs.get('params')} | Body: {kwargs.get('json')}")

        try:
            response = self.client.request(method, endpoint, **kwargs)

            # 2. 检查 HTTP 状态码 (4xx, 5xx 抛出异常)
            response.raise_for_status()

            # 3. 记录响应日志 (截断过长内容)
            resp_json = response.json()
            logger.debug(f"Response: {str(resp_json)[:200]}...")

            return resp_json

        except httpx.HTTPStatusError as e:
            logger.error(f"HTTP Error {e.response.status_code}: {e.response.text}")
            raise
        except httpx.RequestError as e:
            logger.warning(f"Network Error: {e} - Retrying...")
            raise

    # --- 泛型方法：直接返回 Pydantic 对象 ---
    def request_model(self, method: str, endpoint: str, response_model: Type[T], **kwargs) -> T:
        data = self._request(method, endpoint, **kwargs)
        # Pydantic V2 语法
        return response_model.model_validate(data)

    def close(self):
        self.client.close()
```

### 4. 业务接口层 (`client/services/*.py`)

**作用**: 按业务领域组织接口，只关注 Endpoint 和模型转换，不关心底层 HTTP 细节。

**`client/services/user.py`**

```python
from client.core import BaseClient
from schemas import UserDTO, CreateUserRequest, UserListDTO

class UserService(BaseClient):

    def get_user(self, user_id: int) -> UserDTO:
        """获取单个用户，返回强类型对象"""
        return self.request_model(
            "GET", 
            f"/users/{user_id}", 
            response_model=UserDTO
        )

    def create_user(self, body: CreateUserRequest) -> UserDTO:
        """创建用户，入参也是 Pydantic 对象"""
        return self.request_model(
            "POST", 
            "/users", 
            response_model=UserDTO,
            json=body.model_dump() # 转为字典
        )
```

**`client/services/order.py`**

```python
from client.core import BaseClient

class OrderService(BaseClient):
    def get_orders(self, page: int = 1):
        # 简单接口也可以不定义 Model，直接返回 dict
        return self._request("GET", "/orders", params={"page": page})
```

### 5. 统一入口与单例 (`client/__init__.py`)

**作用**: 门面模式 (Facade)，外部只需要引入这一个类。

```python
from client.services.user import UserService
from client.services.order import OrderService

class ThirdPartyAPI:
    def __init__(self):
        # 组合模式：将各个 Service 挂载到主类
        # 注意：这里各个 Service 可能会共享同一个 httpx.Client 实例以复用连接池
        # 为简单起见，这里通过继承分别实例化，实际优化可传入 shared_client
        self.user = UserService()
        self.order = OrderService()

    def close(self):
        self.user.close()
        self.order.close()

# 单例实例
api = ThirdPartyAPI()
```

### 6. 使用示例 (`main.py`)

这就是最终在业务逻辑中调用的样子：**清晰、有类型提示、自动重试、自动日志**。

```python
import sys
from loguru import logger
from client import api
from schemas import CreateUserRequest

# 配置 Loguru 格式
logger.remove()
logger.add(sys.stderr, format="<green>{time:HH:mm:ss}</green> | <level>{level}</level> | {message}")

def main
        # 1. 简单查询
        logger.info("Starting user fetch...")
        user = api.user.get_user(user_id=101)
        # IDE 会在这里提示 user 对象的属性 (.username, .email)
        print(f"Got user: {user.username}")

        # 2. 创建操作 (带参数校验)
        new_user = CreateUserRequest(username="test_dev", password="secret_password")
        created = api.user.create_user(new_user)
        print(f"Created user id: {created.id}")

    except Exception as e:
        logger.exception("Business logic failed")
    finally:
        api.close()

if __name__ == "__main__":
    main()
```

### 🚀 为什么这是“最佳实践”？

1. **Httpx**: 比如 `requests` 更现代化，原生支持 `async`（这套代码稍微改改就能变成异步），支持 HTTP/2。
2. **Pydantic**:
   * **In**: 保证你发给对方的数据是合法的（比如 `username` 不能为空）。
   * **Out**: 保证对方返回的数据结构是你预期的，且让 IDE 能自动补全字段，不再需要猜字典里的 Key。
   * **Settings**: 将配置与代码解耦，符合 [12-Factor App](https://12factor.net/) 原则。
3. **Tenacity**: 网络抖动是常态。把重试逻辑用 `@retry` 装饰器写在底层，业务代码不需要写满 `try-except` 循环。
4. **Loguru**: 极其优雅的日志库，不再需要复杂的 logging 配置，配合底层拦截，能自动记录每一次外部交互，排查 Bug 神器。
5. **结构分层**:
   * `Settings` 管配置。
   * `Schemas` 管数据格式。
   * `Core` 管 HTTP 协议细节。
   * `Service` 管业务接口。
   * **修改任何一层都不会影响其他层。**
