# 🦀 Rust 编译速度优化 Cheatsheet

> 适用于日常开发（Debug / 增量编译）场景，追求最短的 Edit → Compile → Run 循环。

---

## 1️⃣ 更换链接器（效果最显著，必做）

链接是编译最后一步，默认链接器很慢。更换后增量编译提速 **30%~60%**。

### Linux

安装 `mold`：

```bash
# Ubuntu / Debian
sudo apt install mold

# Arch
sudo pacman -S mold

# Fedora
sudo dnf install mold
```

配置 `.cargo/config.toml`：

```toml
[target.'cfg(target_os = "linux")']
rustflags = ["-C", "link-arg=-fuse-ld=mold"]
```

### macOS

macOS 自带的链接器已经较快。如果追求极致，可尝试 `lld`：

```toml
[target.x86_64-apple-darwin]
rustflags = ["-C", "link-arg=-fuse-ld=lld"]

[target.aarch64-apple-darwin]
rustflags = ["-C", "link-arg=-fuse-ld=lld"]
```

### Windows

推荐使用 `lld`（通常随 LLVM / Visual Studio 安装）：

```toml
[target.x86_64-pc-windows-msvc]
linker = "rust-lld.exe"
```

> 💡 **全局生效**：把上述配置写入 `~/.cargo/config.toml`，所有项目自动使用。

## 2️⃣ 使用 Cranelift 后端（Dev 构建提速）

将 Debug 模式的代码生成后端从 LLVM 换成 Cranelift，**牺牲运行时性能换取编译速度**。

```bash
# 安装 Cranelift 组件（需要 nightly）
rustup component add rustc-codegen-cranelift-preview --toolchain nightly
```

在 `Cargo.toml` 中配置：

```toml
[profile.dev]
codegen-backend = "cranelift"
```

运行时需要 nightly + unstable flag：

```bash
cargo +nightly -Z codegen-backend build
```

> ⚠️ **注意**：
>
> - 仅用于 `dev` 构建，`release` 仍应使用 LLVM
> - 生成的代码运行较慢（但开发阶段不在乎）
> - 不支持所有 target 和特性（如 unwinding）

## 3️⃣ 优化 Dev Profile 配置

在 `Cargo.toml` 中对 Debug 构建进行调优：

```toml
[profile.dev]
opt-level = 0          # 不做优化（默认，保持即可）
debug = true           # 保留调试信息
incremental = true     # 确保增量编译开启（默认已开）
codegen-units = 256    # 更多并行编译单元（牺牲运行时性能）
lto = false            # 不做链接时优化（默认，保持即可）

# 让依赖库用稍高的优化级别（依赖不常改，编译一次受益多次）
[profile.dev.package."*"]
opt-level = 1
```

> 💡 `codegen-units = 256`（默认值）已经是 dev 模式最大值，不需要额外设置。

## 4️⃣ 精简依赖特性（Feature Flags）

**永远不要无脑使用 `features = ["full"]`**。

```toml
# ❌ 错误示范：拉入所有功能，编译巨量无用代码
[dependencies]
tokio = { version = "1", features = ["full"] }

# ✅ 正确做法：只开启你用到的功能
[dependencies]
tokio = { version = "1", features = ["rt-multi-thread", "macros", "net"] }
```

常见"胖"依赖瘦身效果：

| 依赖 | `full` 编译时间 | 精简后时间 | 节省 |
|------|----------------|-----------|------|
| `tokio` | ~15s | ~6s | ~60% |
| `reqwest` | ~25s | ~10s | ~60% |
| `serde` | ~4s | ~2s | ~50% |

## 5️⃣ Cargo Workspace 拆分（中大型项目必做）

将单体项目拆分为多个 crate，只有改动的 crate 会重新编译：

```
my-project/
├── Cargo.toml          # workspace 根
├── crates/
│   ├── core/           # 核心业务逻辑（很少改动）
│   │   ├── Cargo.toml
│   │   └── src/
│   ├── db/             # 数据库层（偶尔改动）
│   │   ├── Cargo.toml
│   │   └── src/
│   └── api/            # HTTP 接口层（频繁改动）
│       ├── Cargo.toml
│       └── src/
```

```toml
# 根 Cargo.toml
[workspace]
members = ["crates/*"]
```

> 💡 修改 `api` 层代码时，`core` 和 `db` 不会重新编译，增量编译飞快。

## 6️⃣ 使用 `cargo check` 替代 `cargo build`

日常写代码时，你只需要知道"代码有没有错"，不需要真正生成可执行文件：

```bash
# ❌ 每次都完整编译（生成二进制，耗时长）
cargo build

# ✅ 只做类型检查和借用检查（跳过代码生成，快 2~5 倍）
cargo check
```

## 7️⃣ 后台持续检查工具

让编译在后台自动运行，你保存代码的瞬间就开始检查：

### bacon（推荐）

```bash
cargo install bacon
bacon         # 默认运行 cargo check
bacon run     # 自动运行项目
bacon test    # 自动跑测试
```

### cargo-watch

```bash
cargo install cargo-watch
cargo watch -x check              # 保存即检查
cargo watch -x run                # 保存即运行
cargo watch -x 'test -- --nocapture'  # 保存即测试
```

## 8️⃣ SQLx 离线模式（用 SQLx 必做）

`sqlx::query!` 宏在编译时连接数据库校验 SQL，非常慢。使用离线模式后读取本地缓存：

```bash
# 生成离线查询元数据
cargo sqlx prepare

# 设置环境变量启用离线模式
export SQLX_OFFLINE=true
```

会在项目根目录生成 `.sqlx/` 文件夹，把它提交到 Git 中，CI 也不需要连数据库了。

---

## 9️⃣ sccache 编译缓存（团队/CI 场景）

跨项目、跨机器复用编译产物：

```bash
cargo install sccache

# 设置环境变量（写入 shell profile）
export RUSTC_WRAPPER=sccache
```

效果：

- 切换 Git 分支后重新编译 → **命中缓存，秒级完成**
- CI 流水线多次构建 → **大幅缩短构建时间**
- 团队共享远程缓存（支持 S3/GCS/Redis）

## 🔟 其他实用技巧

### 减少过程宏依赖

```toml
# 如果不需要 derive 宏，手动实现 trait
serde = { version = "1", default-features = false }  # 不自动拉 serde_derive
```

### 使用 `cargo build --timings` 分析瓶颈

```bash
cargo build --timings
# 会生成 target/cargo-timings/cargo-timing.html
# 用浏览器打开，可视化看到每个 crate 的编译时间
```

### 硬件层面

- 🔥 **SSD 是刚需**：Rust 编译是重 I/O 操作，机械硬盘体验极差
- 💾 **内存 ≥ 16GB**：大项目链接阶段内存消耗巨大
- 🧮 **多核 CPU**：Rust 编译天然支持多核并行，核越多越快

## 📋 Quick Setup（一键配置模板）

将以下内容保存为项目的 `.cargo/config.toml`，即可获得开箱即用的优化体验：

```toml
# ===========================================
# Rust 编译速度优化配置
# ===========================================

# --- Linux: 使用 mold 链接器 ---
[target.'cfg(target_os = "linux")']
rustflags = ["-C", "link-arg=-fuse-ld=mold"]

# --- macOS: 使用 lld 链接器（可选） ---
# [target.aarch64-apple-darwin]
# rustflags = ["-C", "link-arg=-fuse-ld=lld"]

# --- Windows: 使用 rust-lld ---
# [target.x86_64-pc-windows-msvc]
# linker = "rust-lld.exe"

# --- 通用优化 ---
[build]
# 并行编译前端 jobs 数（默认为 CPU 核数）
# jobs = 8

[term]
# 彩色输出
color = "auto"
```

配合 `Cargo.toml`：

```toml
[profile.dev]
incremental = true

# 依赖库用 opt-level=1 编译，提升运行时性能且只编译一次
[profile.dev.package."*"]
opt-level = 1
```
