# RTF：Role / Task / Format

RTF 用最少结构锁定“角色、任务、输出格式”，适合快速落地或输出形态严格的场景。

## 适用场景

* 任务目标明确、上下文较少。
* 输出需要严格格式（表格、JSON、清单）。
* 需要快速写出可复用的 Prompt。

## 结构拆解

* Role：模型扮演的角色与能力边界。
* Task：清晰描述要完成的工作。
* Format：输出结构、字段、语言与长度要求。

## 模板

```text
Role: {角色/身份/能力范围}
Task: {要完成的具体任务}
Format: {输出结构 + 语言 + 长度 + 约束}
```

## 示例

```text
Role: 你是资深数据分析师，擅长异常检测。
Task: 基于以下 CSV 数据找出异常值，并解释原因。
Format: 用 Markdown 表格输出，列为「行号、字段、异常原因」，表格下补充 3 条结论。
```

## 注意事项

* Role 不要过泛，最好包含专业领域与边界。
* Task 要避免含糊词（如“分析一下”），改为可验证结果。
* Format 写清楚结构、字段、语言、长度与禁止项。
