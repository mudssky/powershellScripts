# CARE：Context / Action / Result / Example

CARE 通过“结果 + 示例”强化输出一致性，适合对格式与风格敏感的场景。

## 适用场景

* 需要严格输出样式或字段。
* 希望模型参考示例降低偏差。
* 格式合规、模板化内容生产。

## 结构拆解

* Context：背景与素材。
* Action：明确要执行的动作。
* Result：期望的结果/结构/验收标准。
* Example：示例或对照样例。

## 模板

```text
Context: {背景/素材}
Action: {要执行的动作}
Result: {输出结构/质量标准}
Example:
{示例输出或样例}
```

## 示例

```text
Context: 请根据以下日志归纳故障原因。
Action: 总结原因并给出 2 条修复建议。
Result: 输出 1 段原因描述 + 2 条建议，建议以序号列表输出。
Example:
原因：缓存连接超时导致请求失败。
建议：
1. 提升缓存连接池上限。
2. 增加连接重试与告警。
```

## 注意事项

* Example 建议短小且“结构正确”，避免引入噪音。
* 如需多示例，保持同一结构与风格。
