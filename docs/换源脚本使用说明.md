# Package Source 使用说明

`scripts/pwsh/misc/Switch-Mirrors.ps1` 是跨平台 package source 的统一 Stage 1 入口。它负责计划、事务、snapshot、状态、drift 检测和恢复；字体、CLI、Profile 等安装脚本只声明所需 target，不保存镜像地址。

## 网络模式

| 模式 | 行为 | 持久性 |
|---|---|---|
| `Direct` | 默认值。不探测、不换源、不创建事务，也不重置已有自定义 source | 无写入 |
| `China` | 应用受支持的国内镜像，重复执行复用原事务与最初 snapshot | 保持到显式 `Restore` |
| `Auto` | 保留健康的官方源或外部自定义源；仅在官方源连续失败时临时换源 | 调用方必须在 `finally`/`trap` 中恢复 |

`Auto` 不会覆盖未受本仓管理的自定义 source。自定义 source 不可用时返回 `ExternalUnavailable` 和退出码 `10`，由用户决定是否修改。

## 动作

| 动作 | 说明 |
|---|---|
| `Plan` | 只返回将使用的 adapter 和支持状态，不探测、不写入 |
| `Apply` | 创建或复用事务，并应用当前可用 target |
| `Ensure` | 工具安装后，把新出现的 target 补进既有事务 |
| `Status` | 只读查看事务、drift、orphan 和回滚命令 |
| `Restore` | 仅恢复该事务实际修改且未发生 drift 的资源 |

`Apply`、`Ensure`、`Restore` 搭配 `-WhatIf` 时分别降级为只读 `Plan`、`Plan`、`Status`。旧 Docker 调用仍可使用 `-DryRun`。

## 快速开始

默认 Direct 计划，不产生状态目录：

```powershell
pwsh -NoProfile -File ./scripts/pwsh/misc/Switch-Mirrors.ps1 `
  -Action Plan `
  -Mode Direct `
  -Target brew,npm,pnpm `
  -OutputFormat Json
```

应用 China 事务：

```powershell
pwsh -NoProfile -File ./scripts/pwsh/misc/Switch-Mirrors.ps1 `
  -Action Apply `
  -Mode China `
  -Phase Runtime `
  -Target brew,npm,pnpm `
  -TransactionId macos-core-sources `
  -OutputFormat Json
```

工具安装后补应用 npm：

```powershell
pwsh -NoProfile -File ./scripts/pwsh/misc/Switch-Mirrors.ps1 `
  -Action Ensure `
  -Target npm `
  -TransactionId macos-core-sources `
  -OutputFormat Json
```

查看和恢复：

```powershell
pwsh -NoProfile -File ./scripts/pwsh/misc/Switch-Mirrors.ps1 `
  -Action Status `
  -TransactionId macos-core-sources `
  -OutputFormat Json

pwsh -NoProfile -File ./scripts/pwsh/misc/Switch-Mirrors.ps1 `
  -Action Restore `
  -TransactionId macos-core-sources `
  -OutputFormat Json
```

## Target 支持矩阵

| Target | Stage 1 adapter | 当前边界 |
|---|---|---|
| `brew` | `managed-env` | chsrc 只在隔离 HOME 生成值，最终写受管 env，不改 shell rc |
| `rustup` | `managed-env` | 与 Homebrew 使用相同受管 env 机制 |
| `npm`、`pnpm`、`pip`、`go` | `chsrc` | chsrc 应用，原生命令读取当前 source，文件事务负责恢复 |
| `debian`、`ubuntu`、`arch` | `chsrc-system` | 需要提权；修改已知系统文件前 snapshot |
| `docker` | `docker` | 保留 `/v2/` 的 `200/401` 探活、JSON 合并和恢复 |
| `winget` | Stage 1 暂不支持 | 当前只允许 Windows Stage 0 helper 使用结构化 WinGet cmdlets |
| `uv`、`rust` | 暂不支持 | 避免覆盖现有 TOML，等待结构化合并 adapter |
| `nix` | 扩展点 | 由 Nix 任务实现 flake/substituter，不使用 `nix-channel` 代替 |

使用 chsrc 的 adapter 要求 `chsrc >= 0.2.5`。版本不足、命令缺失、权限不足或 adapter 未验证时返回 `Blocked`/`Unsupported`，不会创建事务并假装已换源。

## Stage 0

Stage 0 发生在 PowerShell 7 或 chsrc 尚不可用时，只处理获得平台包管理器、PowerShell 7 和 chsrc 所需的最小网络例外。

### macOS 与 POSIX

`scripts/bash/package-source-bootstrap.sh` 当前只安全支持 Homebrew target。镜像变量仅注入被包装的单个命令，不写 shell rc：

```bash
./scripts/bash/package-source-bootstrap.sh \
  --mode China \
  --target brew \
  -- \
  brew install --cask powershell
```

`Auto` 先探测 Homebrew 官方端点；探测成功时不注入镜像。JSON 决策写入 stderr，避免污染被包装命令的 stdout。

### Windows

`Invoke-PackageSourceBootstrap.ps1` 保持 Windows PowerShell 5.1 语法，只在以下条件全部满足时管理 winget source：

- 当前进程是 Windows 管理员。
- 已安装 `Microsoft.WinGet.Client`，且存在 `Get-WinGetSource`、`Add-WinGetSource`、`Remove-WinGetSource`。
- 原 source 能先保存为结构化 snapshot。

```powershell
powershell.exe -NoProfile -File .\scripts\pwsh\misc\Invoke-PackageSourceBootstrap.ps1 `
  -Mode China `
  -DryRun `
  -OutputFormat Json
```

条件不满足时返回 `Blocked`，不解析 `winget source list` 的表格文本。

Windows `03configureSources.ps1` 只把 winget 的 Stage 0 snapshot/capability 作为只读结果加入单文档 JSON；winget 仍不属于共享 Stage 1 transaction ID。实际 Git、PowerShell、AutoHotkey winget 安装由 `windows/00quickstart.ps1` 或独立 Windows 叶子通过上述 helper 包装。

### 新机执行顺序

平台流水线必须按依赖执行，而不是在 PowerShell 7 缺失时直接调用 Stage 1：

```text
repo/bootstrap
  -> 平台包管理器
  -> Stage 0 包装 PowerShell 7/chsrc 安装
  -> Stage 1 Apply sources
  -> CLI/fonts/profile 等安装
  -> Ensure 后出现的 target
  -> Auto finally Restore 或保留 China rollback 命令
```

macOS 编号脚本应让 PowerShell bootstrap 位于 Stage 1 source 步骤之前。Linux 新机若在 PowerShell 7/chsrc 前需要修改 apt/pacman source，而对应原生 Stage 0 adapter 尚未提供，`China`/`Auto` 必须返回 `Blocked`，不能静默退回官方源并继续。

## Auto 调用合同

根编排器必须保存返回的 transaction ID，并保证清理：

```powershell
$sourceResult = & $switchMirrors `
    -Action Apply `
    -Mode Auto `
    -Target brew,npm `
    -OutputFormat Json | ConvertFrom-Json

try {
    # 执行依赖网络的安装步骤。
}
finally {
    if (-not [string]::IsNullOrWhiteSpace($sourceResult.TransactionId)) {
        & $switchMirrors -Action Restore -TransactionId $sourceResult.TransactionId
    }
}
```

adapter 应用失败时，引擎会立即尝试恢复 Auto 事务。进程崩溃留下的 Auto 事务由下次 `Status` 报告为 `Orphaned`，下一次 Auto Apply 会先恢复；恢复遇到 drift 时返回 `Blocked`。

## 状态与安全

默认状态目录：

- macOS/Linux：`${XDG_STATE_HOME:-$HOME/.local/state}/powershellScripts/package-sources/`
- Windows：`$env:LOCALAPPDATA\powershellScripts\package-sources\`

每个事务保存 `manifest.json`、`snapshots/` 和 `logs/`。manifest 只记录安全元数据和 hash，不保存 npm token、认证配置全文或 secret 环境变量。

Restore 规则：

1. 当前 hash 等于事务的 after hash时，恢复原 snapshot。
2. 当前 hash 已变化时标记 `Drifted`，返回 `10` 并拒绝覆盖。
3. 仅人工调用可加 `-Force`；引擎会先保存 drift 后的当前文件。

## JSON 与退出码

`-OutputFormat Json` 的 stdout 只包含一个 JSON document。逐 target 结果包含 `Target`、`Mode`、`Phase`、`Adapter`、`Status`、`Source`、`Persistent`、`TransactionId`、`Message` 和 `Rollback`。

| 退出码 | 含义 |
|---|---|
| `0` | 成功、Direct、无需变更或只读状态正常 |
| `1` | 执行或验证失败 |
| `2` | 参数错误 |
| `10` | 缺少权限/工具、Unsupported、drift、orphan 或需人工处理 |

## Docker 兼容调用

旧参数仍映射到同一 Docker adapter：

```powershell
./scripts/pwsh/misc/Switch-Mirrors.ps1 -Target docker -UseChinaMirror -DryRun
./scripts/pwsh/misc/Switch-Mirrors.ps1 -Target docker -MirrorUrls 'https://registry-1.docker.io'
./scripts/pwsh/misc/Switch-Mirrors.ps1 -Target docker -Disable
```

真实 China/Auto Apply 会修改用户级或系统级 source。默认 QA 只使用临时 HOME、fixture 和伪命令，不访问或改写本机真实 source。
