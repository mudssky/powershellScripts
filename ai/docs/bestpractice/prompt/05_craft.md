# CRAFT：Context / Role / Action / Format / Tone

CRAFT 对写作与交付形态要求明确，结构轻量、可快速复用。

## 适用场景

* 需要明确行动指令并约束表达方式。
* 需要在风格与格式间取得平衡。
* 适合运营文案、产品说明、知识总结。

## 结构拆解

* Context：背景与素材。
* Role：角色与能力。
* Action：具体要做的动作。
* Format：输出格式与结构。
* Tone：语气与表达偏好。

## 模板

```text
Context: {背景/素材}
Role: {角色/能力}
Action: {要执行的动作}
Format: {输出结构/字段/长度}
Tone: {语气/风格}
```

## 示例

```text
Context: 以下是 3 条用户反馈。
Role: 你是产品经理。
Action: 归纳共性问题并给出改进建议。
Format: 输出 3 条要点，每条不超过 40 字。
Tone: 简洁、理性。
```

## 注意事项

* Action 要“可执行”，避免抽象目标。
* Format 与 Tone 可以写成可复用的“模板片段”。
