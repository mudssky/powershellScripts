# CRISP：Context / Role / Intent / Style / Parameters

CRISP 兼顾上下文、角色与风格，适合需要明确业务背景与写作约束的场景。

## 适用场景

* 有业务背景、需要模型理解上下文。
* 输出风格（语气、受众、表达方式）很重要。
* 需要明确参数与边界（长度、语言、禁止项）。

## 结构拆解

* Context：背景信息、现状、已知条件。
* Role：模型身份与专业能力。
* Intent：任务目标与交付物。
* Style：表达风格、语气、受众偏好。
* Parameters：长度、格式、限制、工具或数据来源。

## 模板

```text
Context: {背景/目标/已有信息}
Role: {角色/身份/领域}
Intent: {要完成的结果}
Style: {语气/风格/受众}
Parameters: {输出格式/长度/限制/引用来源}
```

## 示例

```text
Context: 我们计划发布一份产品更新说明，面向老用户。
Role: 你是资深产品运营，擅长写面向用户的更新公告。
Intent: 输出 1 篇更新说明，突出 3 个核心改进。
Style: 语气友好、简洁，面向非技术用户。
Parameters: 300-400 字，分 3 个小节，避免技术术语。
```

## 注意事项

* Context 里避免堆砌无关信息，保留对任务有贡献的内容。
* Parameters 建议显式写出“禁止项”和“必须项”。
