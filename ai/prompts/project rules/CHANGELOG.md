# 更新日志 (Changelog)

所有关于“项目规则”提示词的重要变更都将记录在此文件中。

## [[v3.0.0]](./03the-rule-architect.md) - 2025-12-25

### 新增

- **智能编辑 (Smart Editing)**: 引入 `Edit over Write` 规则，鼓励对现有文件使用补丁/编辑工具，而不是全量重写，以节省 Token 并减少错误。
- **Agent 模式优化**: 针对 CLI Agent（如 Claude Code, Trae）量身定制规则。
- **强制验证**: 计划模板现在明确要求“自动执行”验证命令。
- **环境感知**: 增加了严禁假设文件存在以及严格的包安全规则。

### 变更

- **拒绝懒惰 (No Laziness)**: 从“全量输出”重新定义为“智能编辑”。现在仅对*新文件*要求全量输出，现有文件应使用 `Edit` 工具。
- **计划模板**: 将“步骤 3：验证”更新为可执行的操作，而不仅仅是列出验证手段。
- **核心宪法 (Core Constitution)**: 彻底重构 `00_core_constitution.md`，使其适配 "Agent Mode"。

## [[v2.0.0]](./deprecrate/02the-rule-architect.md) - 历史版本

### 新增

- 初始 "Rule Architect" 角色定义。
- 4 层规则结构（核心、工作流、编码标准、项目特定）。
- 强制规划要求。
- “拒绝懒惰”规则（强调全量输出）。
