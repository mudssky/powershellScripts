根据官方文档，以下是**基础用法**和**4个非常实用的进阶场景**：

### 1. 基础用法 (快捷键)

最核心的功能就是“标记”和“跳转”。默认快捷键如下（Windows / Mac）：

* **打标签/取消标签 (Toggle)**: `Ctrl+Alt+K` / `Cmd+Option+K`
  * 在当前行打一个蓝色小书签。
* **跳转到下一个 (Next)**: `Ctrl+Alt+L` / `Cmd+Option+L`
* **跳转到上一个 (Previous)**: `Ctrl+Alt+J` / `Cmd+Option+J`

---

### 2. 实用进阶用法

#### 用法一：给书签“起名字” (Labeled Bookmarks)

当你打了很多书签时，很容易忘记哪个是干什么的。

* **操作**：使用命令面板 (`Ctrl+Shift+P` / `Cmd+Shift+P`) 输入 `Bookmarks: Toggle Labeled`。
* **效果**：VS Code 会弹出一个输入框让你写备注（例如：“这里是登录逻辑入口”）。
* **好处**：在侧边栏的书签列表中，你会直接看到这个名字，而不是枯燥的行号。

#### 用法二：日志分析神器 (Selection Features)

这是官方文档特别推荐的一个功能，非常适合处理**长日志文件**或**代码重构**。

* **场景**：你有一个几千行的 Log 文件，你想把所有带有 "ERROR" 的行找出来并复制走。
* **操作**：
    1. 手动或用搜索功能，在所有感兴趣的行打上书签。
    2. 运行命令 `Bookmarks: Select Lines`。
* **效果**：所有被打书签的行会被**同时选中**（多光标模式）。你现在按 `Ctrl+C`，就能一次性把这些分散的行复制出来。

#### 用法三：跨文件/项目概览 (Side Bar)

安装插件后，左侧活动栏会多出一个**书签图标**。

* **功能**：它会以树状图显示当前项目中所有文件的书签。
* **实用性**：
  * 可以右键点击书签进行重命名。
  * 即使文件关闭了，点击列表也能立刻打开文件并跳到那一行。
  * 适合作为阅读大型源码时的“阅读路径”记录。

#### 用法四：团队共享书签 (Save in Project)

默认情况下，书签是保存在你的本地 VS Code 里的，关闭项目再打开还在，但同事看不到。

* **设置**：在设置中开启 `bookmarks.saveBookmarksInProject: true`。
* **效果**：书签信息会被保存到项目根目录的 `.vscode/bookmarks.json` 文件中。
* **实用性**：你可以把这个文件提交到 Git。当同事拉取代码后，也能看到你在关键代码位置打的标记（例如：“这里是核心算法，新人必看”）。

---

### 3. 推荐设置 (Settings)

为了更好用，建议在设置 (`Ctrl+,`) 中调整以下选项：

* **`Bookmarks: Navigate Through All Files`** (默认开启):
  * 建议保持开启。这样按“跳转下一个”时，如果当前文件没了，它会自动跳到下一个有书签的文件，而不是卡住。
* **`Bookmarks: Wrap Navigation`**:
  * 建议开启。跳到最后一个书签后，再按一下会自动回到第一个，形成循环。

### 总结

如果只是简单的“看一眼回头再看”，用 `Ctrl+Alt+K` 打标就够了；如果是为了**阅读源码**或**分析日志**，请务必配合**侧边栏 (Side Bar)** 和 **多行选择 (Select Lines)** 功能使用。
